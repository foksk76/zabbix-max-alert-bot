# Zabbix MAX Alert Bot

Небольшой проект для доставки уведомлений из Zabbix в корпоративный мессенджер МАХ.

Смысл простой: события мониторинга должны быстро доходить до тех, кто за них отвечает или заинтересован в их получении. Рабочий канал Telegram остается без изменений. МАХ добавляется как второй канал доставки, чтобы повысить надежность оповещений для ответственных и заинтересованных получателей.

На первом этапе проект не пытается быть большой платформой, SIEM-интеграцией или AI-помощником. Это аккуратная прикладная интеграция Zabbix → МАХ для отправки уведомлений о проблемах, инцидентах, сбоях и восстановлении сервисов.

## Задача первого этапа

Реализовать отдельный Media type в Zabbix с типом `Webhook`, который отправляет сообщения в MAX Bot API.

Целевая схема:

```text
Zabbix Action
  ├─ Telegram Webhook
  └─ MAX Webhook
       └─ MAX Bot API
            └─ чат ответственных или заинтересованных получателей
```

Так одно событие мониторинга может быть доставлено сразу в два независимых канала: Telegram и МАХ.

## Что входит

- создание и настройка бота в МАХ;
- создание Media type `MAX` в Zabbix;
- настройка webhook-скрипта;
- определение формата уведомлений в зависимости от необходимого уровня информативности для получателя;
- маршрутизация уведомлений по чатам, группам или отдельным получателям;
- проверка доставки тестовых событий из Zabbix;
- фиксация минимальной эксплуатационной документации.

## Что не входит в первый этап

- интеграция с SIEM;
- AI-обработка событий;
- автоматическая triage-логика;
- подтверждение инцидентов из МАХ;
- управление событиями Zabbix из мессенджера;
- автоматическое реагирование.

Эти вещи можно развивать позже, когда базовый транспорт уведомлений будет проверен и принят в эксплуатацию.

## Основной файл

Первый рабочий артефакт проекта:

```text
src/zabbix-media-type/max-webhook.js
```

Это JavaScript-скрипт для Zabbix Media type `Webhook`. Его можно вставить в поле `Скрипт` при создании нового способа оповещения в Zabbix.

## Базовые параметры Media type

Базовые параметры для настройки Zabbix Media type описаны в файле:

```text
docs/zabbix-media-type.md
```

`To` используется как идентификатор нужного чата или получателя в МАХ. Для разных групп сопровождения, мониторинга, реагирования или отдельных получателей можно использовать разные значения `To` через пользователей, группы пользователей и Actions в Zabbix.

## Ожидаемый результат

Ответственные и заинтересованные получатели получают оперативные уведомления из Zabbix в МАХ.

Надежность доставки оповещений системы мониторинга ответственным за сервисы повышается до целевого уровня 100% за счет дублирования уведомлений в двух независимых каналах — Telegram и МАХ.

## Статус

Проект находится на первом этапе: подготовка базовой интеграции Zabbix → МАХ без SIEM и без AI-логики.
