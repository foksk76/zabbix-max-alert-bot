# Documentation policy

Документация проекта ведется по принципу `documentation-and-adrs`: фиксируем не только что сделано, но и почему выбран именно такой вариант.

## Цель

Сделать так, чтобы будущий инженер или AI-агент мог быстро понять:

- текущую границу проекта;
- какие решения уже приняты;
- какие варианты были отклонены;
- где менять настройки;
- какие изменения требуют ADR.

## Каноничные места

```text
README.md                         краткое описание проекта и быстрый вход
docs/                             эксплуатационная и проектная документация
docs/decisions/                   ADR и история решений
CHANGELOG.md                      заметные изменения
AGENTS.md                         правила для AI-агентов
.agents/project-context.md         короткий рабочий контекст для агента
```

## Когда обновлять документацию

Документацию нужно обновлять, если меняется:

- поведение webhook;
- формат параметров Zabbix Media type;
- граница текущего этапа;
- процесс разработки;
- набор внешних skills;
- правила безопасности;
- порядок проверки или эксплуатации.

## Когда создавать ADR

ADR нужен, если:

- выбирается один из нескольких вариантов реализации;
- решение будет трудно или дорого менять позже;
- меняется архитектура;
- появляется новый внешний инструмент или зависимость;
- меняется место хранения проектных решений;
- меняется подход к AI-assisted разработке.

ADR создается в:

```text
docs/decisions/
```

## Как писать

Хорошая документация отвечает на вопросы:

- зачем это нужно;
- какие ограничения были учтены;
- что решили;
- какие альтернативы отклонили;
- какие последствия у решения.

Не нужно документировать очевидный код. Нужно документировать причины, ограничения и известные ловушки.

## Для AI-агентов

Перед изменением документации агент должен проверить:

1. `AGENTS.md`;
2. `.agents/project-context.md`;
3. `docs/decisions/README.md`;
4. текущий профильный документ в `docs/`.

Если агент собирается изменить архитектуру или границы проекта, сначала создается ADR, затем меняются остальные файлы.
